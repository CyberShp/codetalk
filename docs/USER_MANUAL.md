# CodeTalk 用户使用手册

> 版本 2.0 · 2026-07-01

CodeTalk 是面向研发、测试、架构和代码审查团队的代码理解与测试分析工作台。当前产品围绕工作空间、智能体编排、AI 线程和覆盖率分析展开，默认本机端口为前端 `3003`、后端 API `3004`。

## 1. 访问与启动

本机部署推荐先打开部署器：

```text
http://127.0.0.1:9000
```

部署完成并启动服务后，默认打开：

```text
http://127.0.0.1:3003
```

后端健康检查地址：

```text
http://127.0.0.1:3004/health
```

如果部署器里修改过端口，以启动页展示的地址为准。

## 2. 当前导航

左侧导航栏可以折叠，当前包含这些入口：

| 入口 | 用途 |
|---|---|
| 工作台 | 查看产品首页、进入主要分析工作流 |
| 工作空间 | 创建和打开本机代码仓库工作空间 |
| 智能体编排 | 运行 module analysis、资源泄漏排查、影响面审查和黑盒测试生成等工作流 |
| AI 线程 | 围绕选定工作空间进行源码优先的连续问答 |
| 覆盖率分析 | 上传覆盖率或测试数据，检查入口发现和补充测试建议 |
| 设置 | 配置模型、Agent provider、系统审计和探测项 |

## 3. 工作空间

进入“工作空间”后，可以创建一个指向本机代码目录的工作空间。路径必须是真实存在且当前用户可读的目录，例如：

```text
/Volumes/Media/dpdk/spdk
```

工作空间详情页会展示索引状态、材料、报告、源码证据、聊天入口和分析入口。增强组件不可用时，页面应显示降级说明，而不是阻断基础浏览。

## 4. AI 线程

AI 线程适合做连续代码阅读和测试分析。选择工作空间后提问时，系统应优先使用工作区源码、输入文件和材料作为证据，再调用配置好的模型生成回答。

生成过程中，诊断、思考或工具输出默认折叠，不应混进最终回答。对话内容很长时，中间阅读区独立滚动；用户向上阅读旧消息时，新流式内容不应强行把滚动位置拉回底部。

## 5. 智能体编排

智能体编排页用于运行结构化工作流。常见工作流包括：

| 工作流 | 适用场景 |
|---|---|
| module_analysis | 梳理模块范围、关键文件、证据卡和报告 |
| resource_leak_hunt | 查找资源申请、释放、错误路径和测试钩子 |
| patch_impact_review | 基于补丁输入分析影响面和流程变化 |
| mr_blackbox_test | 从变更或分析结果生成外部可执行的黑盒测试用例 |

运行工作流前应填写目标模块、补丁或补充说明。结果页会展示 artifact、validation、acceptance audit 和可下载内容。

## 6. 覆盖率分析

覆盖率分析页用于导入覆盖率或测试执行数据。系统会尝试发现真实入口、评估 black-box readiness 和 gray-box required 比例，并给出覆盖不足入口的补充测试建议。

格式错误的文件应显示明确修复建议；分析 artifact 与页面摘要应保持一致。

## 7. 设置与模型

设置页用于维护模型和 Agent provider。配置模型时需要填写 API 类型、Base URL、API Key 和模型名。保存或测试连接时，页面、日志和导出内容都不应显示完整密钥。

外部 Agent provider 可用于本机智能体编排。提供方输出乱码、数字进度、诊断或 reasoning 事件时，CodeTalk 应进行过滤和折叠，只把可读结论放入最终回答。

## 8. 常见问题

### 首页打不开

先看部署器启动页里 frontend 是否运行。默认前端端口是 `3003`，如有冲突请在部署器里改端口后重新启动。

### 后端不可用

确认 backend 是否运行，并访问：

```text
http://127.0.0.1:3004/health
```

正常会返回：

```json
{"status":"ok"}
```

### AI 没有读源码

确认当前 AI 线程已经选择工作空间，且工作空间路径存在。问题中可以明确要求“基于工作区源码和材料回答，并列出文件证据”。如果仍没有证据，应检查 provider 探测和工作空间索引状态。

### 页面卡顿

优先检查是否正在渲染超长回答、超大 SFMEA 表或大量 artifact。当前 UI 已减少持续动画和长列表入场动画；若仍卡顿，应保留浏览器 trace、控制台日志和任务 artifact 继续定位。
