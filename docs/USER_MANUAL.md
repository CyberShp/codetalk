## 1. 产品简介

CodeTalks 是一个**代码分析编排与可视化平台**。它本身不执行任何分析逻辑，而是统一调度 5 个独立的开源分析工具，将结果聚合展示在一个界面中。

**核心能力一览**：

| 你想做什么 | CodeTalks 帮你调度的工具 | 对应页面 |
|-----------|------------------------|---------|
| 浏览自动生成的项目文档 | DeepWiki | 仓库详情 → 文档 |
| 查看代码知识图谱（函数调用、类继承、文件依赖） | GitNexus | 仓库详情 → 神经图谱 |
| 全文搜索/正则搜索代码 | Zoekt / GitNexus Grep | 仓库详情 → 搜索 |
| 安全扫描（漏洞发现） | Semgrep | 静态分析 → 发现 |
| 控制流/数据流分析 | Joern CPG | 静态分析 → 分支 / 污点 |
| 自动生成测试用例 | Joern + Semgrep + AI | 静态分析 → 测试点 |
| 向 AI 提问代码细节 | DeepWiki RAG | AI 问答 |

**工作流概览**：

```
创建项目 → 添加仓库 → 同步代码 → 创建分析任务 → 查看结果
```

## 2. 系统要求与访问

### 浏览器要求

- Chrome 90+ / Edge 90+ / Firefox 88+（推荐 Chrome）
- 最低分辨率 1280×720，推荐 1920×1080

### 访问地址

| 服务 | 默认端口 | 示例地址 |
|------|---------|---------|
| CodeTalks 前端 | 3000 | `http://your-server:3000` |
| CodeTalks API | 8000 | `http://your-server:8000` |

## 3. 界面总览

打开 CodeTalks 后，你会看到左侧固定导航栏和右侧内容区域。

### 左侧导航栏

导航栏宽 64px，始终可见，包含以下入口：

| 图标 | 名称 | 说明 |
|------|------|------|
| 📊 | **仪表盘** | 系统概览、项目统计、运行中任务 |
| 📋 | **任务** | 所有分析任务的列表与管理 |
| 🔧 | **工具** | 查看 5 个分析工具的在线状态 |
| 📁 | **资产** | 管理项目和代码仓库 |
| ⚙️ | **设置** | 配置 LLM 模型、查看系统健康 |

导航栏底部有一个 **「新建分析」** 快捷按钮，可从任何页面快速创建分析任务。

### 主题风格

CodeTalks 采用深色主题（Kinetic Shadow），页面背景为深蓝黑色（#10141A），配合毛玻璃效果面板。三种强调色：
- 青蓝（#A4E6FF）：主要操作、链接
- 浅绿（#ECFFE3）：成功状态
- 浅红（#FFD1CD）：警告、错误

## 4. 仪表盘

路径：**首页 / 仪表盘**

仪表盘是登录后的第一个页面，展示全局概览。

### 顶部统计卡片

四张卡片横排展示核心指标：

| 卡片名 | 含义 |
|--------|------|
| **项目总数** | 你创建的项目数量 |
| **运行中任务** | 当前正在执行的分析任务数 |
| **已完成** | 历史已完成的任务数 |
| **工具健康** | 在线工具数 / 总工具数（如 "5/5"） |

### 最近活动

左下方面板列出最近 5 条任务记录，每条显示：
- 任务 ID（前 8 位）
- 任务类型（如 gitnexus_analysis、semgrep_scan）
- 使用的工具（彩色标签）
- 状态徽章：运行中（蓝）/ 已完成（绿）/ 失败（红）/ 等待中（灰）

### 运行中任务

右下方面板展示当前所有正在执行的任务，带进度条和百分比。此面板每 5 秒自动刷新。

### 异常情况

- **后端不可达**：页面显示错误提示，带「重试」按钮
- **部分加载失败**：显示 "部分数据加载失败" 黄色提示条，已加载的数据正常展示

## 5. 资产管理

路径：**导航栏 → 资产**

资产页面用于管理你的项目和代码仓库。采用左右分栏布局：
- **左栏**：项目列表
- **右栏**：选中项目的仓库表格

---

### 5.1 创建项目

1. 点击页面右上角 **「新建项目」** 按钮
2. 在弹窗中填写：
   - **项目名称**（必填）
   - **描述**（可选）
3. 点击 **「创建」**

创建成功后项目自动出现在左栏，并被选中。

---

### 5.2 添加仓库

1. 先在左栏选中一个项目
2. 点击右栏上方的 **「添加仓库」** 按钮
3. 选择代码来源类型：

#### Git URL 方式

适用于有可访问的 Git 仓库地址的场景。

| 字段 | 说明 | 示例 |
|------|------|------|
| 仓库名称 | 自定义名称 | `my-backend` |
| Git URL | 仓库克隆地址 | `https://gitlab.internal/team/backend.git` |
| 分支 | 要分析的分支，默认 `main` | `develop` |

#### 本地路径方式

适用于代码已存放在服务器共享目录的场景。

| 字段 | 说明 | 示例 |
|------|------|------|
| 仓库名称 | 自定义名称 | `legacy-service` |
| 路径 | `shared-repos` 目录下的相对路径 | `team-a/legacy-service` |

> **注意**：本地路径指的是服务器 `shared-repos` 挂载目录下的路径，不是你本机的路径。具体的共享目录位置请咨询运维。

4. 点击 **「添加」**

---

### 5.3 同步仓库

添加仓库后，需要先同步代码才能执行分析。

1. 在仓库列表中找到目标仓库
2. 点击右侧 **「同步」** 按钮
3. 按钮变为旋转图标，表示正在同步
4. 同步完成后，**「最近索引」** 列更新为当前时间

**同步做了什么？**
- Git URL 方式：后端执行 `git clone` 或 `git pull`
- 本地路径方式：验证路径是否存在，建立索引

**同步失败处理**：
- 页面顶部出现红色错误条
- 常见原因：仓库地址无法访问、认证失败、本地路径不存在
- 检查地址后重新点击「同步」

**取消同步**：同步进行中时，按钮变为 **「取消」**，点击可中断。

---

### 5.4 删除项目/仓库

**删除仓库**：
1. 鼠标悬停到仓库行右侧的垃圾桶图标
2. 点击后弹出确认框
3. 确认删除 — **所有相关的分析任务将一同删除，不可撤销**

**删除项目**：
1. 鼠标悬停到左栏项目名称右侧，出现垃圾桶图标
2. 点击后弹出确认框
3. 确认删除 — **项目下所有仓库和任务将一同删除，不可撤销**

## 6. 仓库详情

路径：**资产 → 点击仓库名称**

仓库详情页是 CodeTalks 的核心页面，提供三个页签：

| 页签 | 功能 | 依赖工具 |
|------|------|---------|
| **文档** | 自动生成的项目文档 + AI 侧边问答 | DeepWiki |
| **神经图谱** | 交互式代码知识图谱 | GitNexus |
| **搜索** | 全文搜索和正则搜索 | Zoekt / GitNexus |

页面顶部固定：
- **返回按钮**（← 资产）
- 仓库名称和分支标签
- **「静态分析」** 按钮 → 跳转到分析页面
- **「全屏 AI 问答」** 按钮 → 跳转到全屏聊天页

---

### 6.1 文档页签

文档页签展示由 DeepWiki 自动生成的项目文档。

#### 左侧：文档内容

- 渲染后的 Markdown 文档，包含标题层级、代码高亮、表格等
- 左侧可折叠 **目录面板**（点击 `[<]` 图标收起/展开）
- 文档内引用了源代码的位置会显示为 **引用链接**（点击可预览文件）
- 顶部有 **刷新按钮** — 重新生成文档

**文档状态说明**：

| 状态 | 显示内容 | 说明 |
|------|---------|------|
| 尚未分析 | "尚未执行 GitNexus 分析。请先创建分析任务。" | 需要先在「任务」页创建 GitNexus 分析任务 |
| 生成中 | "生成中…" + 进度指示 | DeepWiki 正在生成文档，请等待 |
| 已生成 | 正常显示文档内容 | 可正常阅读和交互 |
| 刷新中 | 进度指示 | 正在重新生成，完成后自动更新 |

#### 右侧：AI 侧边聊天

点击页面右上方 **「侧边 Chat」** 按钮可打开/关闭侧边聊天面板。

聊天面板功能：
- 顶部显示 **Neural Link** 品牌名
- 显示当前上下文范围：「N files in scope」（文档关联文件数）或「GLOBAL_CONTEXT」
- 输入框在底部，支持 Enter 发送
- 对话自动保存，刷新页面不会丢失
- 关闭面板后再打开，历史对话仍在

**使用建议**：在阅读文档时，如果对某个部分有疑问，可以打开侧边 Chat 直接提问，AI 会结合当前文档上下文回答。

---

### 6.2 神经图谱页签

神经图谱展示仓库的代码结构关系图——函数调用、类继承、文件导入等关系一目了然。

#### 符号搜索栏（顶部）

| 组件 | 说明 |
|------|------|
| 搜索模式切换 | **Hybrid**（混合，推荐）/ **BM25**（关键词）/ **Semantic**（语义） |
| 搜索输入框 | 输入函数名、类名或文件名 |
| 搜索结果列表 | 显示匹配节点的名称、类型、分数、连接数 |

操作流程：
1. 在搜索框输入你想找的符号名（如 `handleRequest`）
2. 搜索结果实时显示
3. 点击某条结果 → 图谱自动定位到该节点

#### 图谱区域（中央）

这是一个可交互的力导向图。

**节点颜色含义**：

| 颜色 | 节点类型 |
|------|---------|
| 绿色 | Function（函数） |
| 紫色 | Class（类） |
| 蓝色 | File（文件） |
| 其他 | Struct、Process、Community 等 |

**边标签含义**：

| 标签 | 关系类型 |
|------|---------|
| CALLS | 函数调用 |
| IMPORTS | 模块导入 |
| EXTENDS | 类继承 |
| IMPLEMENTS | 接口实现 |
| CONTAINS | 文件包含 |

**交互操作**：

| 操作 | 效果 |
|------|------|
| 单击节点 | 右侧面板显示节点详情 |
| 拖拽空白区域 | 平移画布 |
| 滚轮 | 缩放 |
| 底部过滤按钮 | 按节点类型筛选显示/隐藏 |

#### 节点详情面板（右侧）

点击图谱中的节点后，右侧展开详情面板：

- **函数/类节点**：
  - 节点名称和类型
  - 关联节点列表（调用者、被调用者、成员）
  - 文件位置和行号范围
  - **「在 Chat 中追问」** 按钮 — 打开浮动聊天窗口，上下文自动关联到这个节点的代码

- **其他节点**：
  - 语法高亮的源代码
  - 文件路径和行号

### 6.3 搜索页签

搜索页签支持两种模式在仓库代码中搜索。

#### 搜索模式

| 模式 | 引擎 | 适用场景 |
|------|------|---------|
| **全文** | Zoekt | 搜索确切的字符串或单词 |
| **正则** | GitNexus Grep | 用正则表达式搜索模式 |

#### 操作步骤

1. 选择搜索模式（全文 / 正则）
2. 在搜索框输入查询内容
3. 按 Enter 或点击搜索按钮
4. 查看结果

#### 搜索结果

结果按文件分组展示：

- 顶部显示匹配文件数和搜索词
- 每个文件可折叠/展开
- 文件内显示每条匹配：
  - 左侧：行号
  - 右侧：代码内容（搜索词高亮）

## 7. 运行时风险分析

路径：**仓库详情 → 点击「静态分析」按钮**
或者：**仓库详情页 URL 后加 `/analysis`**

运行时风险分析页面整合了 Semgrep 和 Joern 两个分析引擎的结果，**重点关注软件在真实运行环境中可能遇到的问题**——异常分支处理、边界值溢出、数值翻转、资源泄漏等运行时风险点，而非单纯的代码规范检查。

### 页面布局

- **左侧导航栏**（208px）：5 个子页签 + 过滤器
- **右侧内容区**：对应页签内容
- **顶部固定**：返回链接 + **「Re-scan」** 按钮

### Re-scan 按钮

| 状态 | 显示 | 说明 |
|------|------|------|
| 空闲 | Re-scan | 点击触发新一轮扫描 |
| 扫描中 | Scanning... | 正在扫描，按钮禁用 |
| 完成 | Done | 扫描成功完成 |

### 左侧过滤器

**风险等级筛选**：
- High Risk（红色）/ Medium Risk（橙色）/ Low Risk（蓝色）
- 点击切换开关，选中时显示 ✕
- 可同时选中多个级别

**分类筛选**：
- 根据扫描结果动态生成（如 boundary、overflow、resource-leak）
- 操作方式与等级筛选相同

---

### 7.1 概览面板

概览是分析页面的默认视图。

#### 工具状态条

顶部显示两个引擎状态：
- **Joern CPG**：绿点 = 在线 / 灰点 = 离线
- **Semgrep**：绿点 = 在线 / 灰点 = 离线

#### 四项统计

| 指标 | 含义 |
|------|------|
| Risk Points | 发现的风险点总数 |
| High Risk | 高危风险数（需立即处理） |
| Medium Risk | 中危风险数（需评审确认） |
| Low Risk | 低危风险数（持续监控） |

#### 风险度分布图（左）

环形图展示三个级别的风险点占比，附带图例和数值。

#### 风险类别图（右）

柱状图展示排名前 5 的风险类别及其数量（如异常分支、边界越界、数值溢出等）。

#### CPG 智能引擎卡片

显示 Joern CPG 引擎的状态（在线/离线）和能力列表。

**无发现时**：显示绿色成功消息 "No Runtime Risks Detected"，表示未检测到运行时风险。

---

### 7.2 风险点列表

风险点列表展示所有扫描发现的运行时风险，支持筛选和展开查看详情。

#### 折叠状态（每条一行）

- 风险级别徽章（High Risk / Medium Risk / Low Risk）
- 规则 ID（标识触发该风险的检测规则）
- 简短描述（该风险在运行时可能导致的问题）
- 悬停显示文件路径和行号

#### 展开后详情

点击任一条目展开，查看完整信息：

| 区域 | 内容 |
|------|------|
| **位置** | 文件路径 + 行号范围 |
| **代码片段** | 语法高亮的问题代码，左侧带彩色标注条 |
| **传播路径** | Source（绿点）→ 中间变量 → Sink（红点）的数据传播链 |
| **修复建议** | Remediation Blueprint — 建议的修复方案 |
| **标签** | CWE 编号等分类标签 |

---

### 7.3 分支分析

分支分析是**运行时风险的核心功能**，利用 Joern CPG 引擎查询函数的控制流分支结构，定位异常分支、缺失的错误处理、不充分的边界检查。

#### 使用步骤

1. 在输入框中输入函数名（如 `iscsi_login`、`handleRequest`）
2. 点击 **「查询」** 按钮
3. 查看结果

#### 结果分三部分

| 部分 | 说明 | 关注点 |
|------|------|--------|
| **控制流分支** | 函数内的 if/else/switch 分支结构 | 是否有未覆盖的分支？是否有异常路径没有处理？ |
| **异常处理路径** | try/catch/finally 等错误处理分支 | 错误处理是否完善？异常是否被静默吞掉？ |
| **边界值比较** | 数值比较、范围检查等边界条件 | 边界值是否正确？是否存在 off-by-one？是否可能溢出？ |

每部分显示条目数量徽章，内容可折叠。

> **前提条件**：需要先运行过 Joern 分析任务，且 Joern 容器在线。

---

### 7.4 数据传播追踪

数据传播追踪（Taint Analysis）追踪数据在函数间的异常传播路径，发现数值翻转、未校验传播、越界传播等运行时风险。

#### 预设按钮

页面顶部提供 4 种常见运行时风险模式的一键预设：

| 预设 | Source 模式 | Sink 模式 | 检测风险 |
|------|-----------|-----------|---------|
| **Overflow** | `read\|recv\|input` | `add\|mul\|sub\|shift` | 数值溢出/翻转 |
| **NullDeref** | `malloc\|calloc\|alloc` | `deref\|memcpy\|strcpy` | 空指针解引用 |
| **Boundary** | `read\|recv\|argc` | `array\|index\|offset` | 数组/缓冲区越界 |
| **Resource** | `open\|fopen\|socket` | `close\|fclose\|free` | 资源泄漏 |

#### 自定义分析

除了预设外，你也可以自行填写：
- **Source Pattern**：数据入口的函数/方法名模式
- **Sink Pattern**：数据到达的目标操作模式

然后点击 **「Trace Dataflow」** 按钮。

#### 结果展示

- 顶部显示 **"N Reachable Paths Found"**（N 条可达路径）
- 每条路径展示为纵向传播链：
  - **Source（绿色圆点）**：数据来源点 + 代码片段 + 文件行号
  - **中间节点**：经过的变量和函数
  - **Sink（红色圆点）**：到达的操作点 + 代码片段 + 文件行号

**无结果时**：显示绿色消息 "No Anomalous Propagation Detected"。

> **前提条件**：同分支分析，需要 Joern 在线且已运行分析任务。

## 12. 常见问题

### Q1：刚添加仓库，点击进去什么都看不到？

**原因**：仓库需要先 **同步** 再 **创建分析任务** 后才有数据。

完整流程：
1. 资产 → 找到仓库 → 点击「同步」，等待完成
2. 点击「分析」按钮，创建 GitNexus 分析任务
3. 任务页面等待任务完成（状态变为绿色「已完成」）
4. 回到仓库详情 → 文档/图谱/搜索即可使用

### Q2：文档页面显示"尚未执行 GitNexus 分析"？

需要先创建一个 GitNexus 分析任务。在仓库详情页点「静态分析」，或在「任务」页新建分析。

### Q3：静态分析页面全是空的，没有发现？

两种可能：
1. **还没扫描**：点击右上角「Re-scan」按钮触发 Semgrep 扫描
2. **项目确实安全**：如果 Re-scan 完成后显示 "Environment Secure"，恭喜，没有发现问题

### Q4：分支分析/污点追踪报错 "服务不可用"？

Joern 工具可能离线。去「工具」页面检查 Joern 的状态：
- 如果显示灰色（离线），联系运维重启 Joern 容器
- 重启后可能需要重新运行分析任务来恢复 CPG 工作空间

### Q5：神经图谱是空白的？

检查：
1. GitNexus 工具是否在线（去「工具」页面查看）
2. 是否已运行过 GitNexus 分析任务（去「任务」页面检查）
3. 如果 GitNexus 容器重启过，需要重新运行分析任务

### Q6：AI 聊天没有回复？

检查：
1. 设置 → LLM 配置中是否有配置且标记为默认
2. 点击「测试」验证 LLM 配置可用
3. 如果使用内网模型，确认模型地址和端口可访问

### Q7：全文搜索没有结果但正则搜索有？

两种搜索使用不同引擎：
- 全文搜索依赖 **Zoekt** 索引，需要先同步并建立索引
- 正则搜索依赖 **GitNexus** 的 grep 功能

去「工具」页面检查 Zoekt 是否在线。

### Q8：同步失败提示 "仓库不存在" 或 "认证失败"？

- **Git URL 方式**：检查 URL 是否正确，仓库是否需要认证（内网 GitLab 可能需要 access token）
- **本地路径方式**：确认文件已放在服务器的 `shared-repos` 目录下

### Q9：数据全部丢失，仪表盘显示 0/0/0？

可能是 PostgreSQL 数据库服务异常。联系运维检查数据库容器状态。正常恢复后数据会自动回来。页面上的「重试」按钮可以手动刷新数据。

### Q10：测试点生成失败？

测试点生成需要同时满足：
1. Semgrep 有扫描结果（已发现问题）
2. Joern CPG 在线
3. LLM 配置可用（用于 AI 生成）

缺少任一条件都可能导致生成失败或结果不完整。
